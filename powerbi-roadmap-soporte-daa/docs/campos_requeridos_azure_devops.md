# Campos requeridos/recomendados en Azure DevOps

## Mínimos
- ID: llave del work item.
- Work Item Type: Feature/User Story/Task.
- Title: texto operativo.
- State: estado base.
- Assigned To: ownership.
- Tags: clasificación.

## Recomendados
- Parent: reconstrucción jerárquica robusta.
- Area Path: segmentación organizacional.
- Iteration Path: seguimiento por sprint/ciclo.
- Created Date: antigüedad.
- Changed Date: actividad reciente.
- Closed Date: cierre/cycle time.
- Priority: priorización de backlog.
- Board Column/Done: estado real de tablero.
- Due Date/Target Date: roadmap temporal real.
